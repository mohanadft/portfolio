# Product

## Register

brand

## Users

Hiring managers and technical recruiters evaluating backend engineering candidates. They arrive from a LinkedIn profile or GitHub link, spend 30-90 seconds scanning, and need to leave with the conviction that this person is worth interviewing. Secondary audience: fellow engineers discovering the work through open-source contributions or technical writing.

## Product Purpose

A portfolio that functions as proof-of-craft. Not a resume with links; a demonstration that the person who built the backend systems also sweats the details on presentation. Success: a recruiter bookmarks the site and forwards it to a hiring manager with "check this out."

## Brand Personality

Sharp, technical, confident. The voice of a backend engineer who speaks precisely, ships reliably, and doesn't need decoration to impress. Terminal aesthetic is an identity choice: it says "I live in the command line" without being gimmicky.

## Anti-references

- Generic dev portfolio templates (shadcn cards, "hire me" hero, identical project grids)
- Over-designed agency sites (gratuitous parallax, 3D blobs, style over substance)
- Plain GitHub README rendered as a site (no craft, no intention)
- Cookie-cutter terminal themes that all look the same (green-on-black with no personality)

## Design Principles

1. **Show, don't decorate.** Every visual element should communicate something. No ornament for ornament's sake.
2. **Precision is personality.** Tight spacing, sharp typography, deliberate color use. The craft IS the message.
3. **Confident restraint.** Bold where it matters, quiet everywhere else. Not safe; selective.
4. **Depth over breadth.** Fewer sections with more substance beats a long scroll of shallow cards.
5. **Technical authenticity.** Terminal metaphors should feel native, not cosplay. Use them where they add meaning, drop them where they don't.

## Accessibility & Inclusion

WCAG AA compliance: 4.5:1 contrast ratio minimum for body text, 3:1 for large text and UI components. Keyboard navigation for all interactive elements. Reduced-motion media query respected. Screen reader support with semantic HTML and ARIA labels where needed.
